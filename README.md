# Detection of Anomalies and Unfair Competition in Commercial Tenders Using NLP

NLP-конвейер для детекции аномалий и признаков ограничения конкуренции в
коммерческой тендерной документации 223-ФЗ.

---

## Финальное состояние (2026-05-10)

**Производственная модель** — каскадный конвейер `rule_detector_v2 + FT
gpt-4.1 v3a с фильтрацией по меткам`:

| Метрика на отложенной выборке (20 лотов) | Каскад | FT v3a (отд.) | gpt-5.5 zero-shot | v1 baseline |
|---|---|---|---|---|
| Нестрогая точность на уровне флага | **97.7 %** | 91.1 % | — | 13.2 % |
| Нестрогая F1 на уровне лота | **90.9 %** | 50.0 % | 61.5 % | — |
| Специфичность | **93.3 %** | 73.3 % | — | 5 % |
| Стоимость / 200 лотов | **$15** | $15 | $80 | $2.50 |

Подробные метрики (включая calibration confidence, secondary annotation 13
LOW-лотов, разбор по меткам) — в `docs/finetune_v3a_results.md` и
`docs/ftv3a_honest_assessment.md`.

### Ключевые научные результаты

1. **Методологический разрыв EU/OECD ↔ 223-ФЗ.** Codebook v1, обоснованный
   Директивой 2014/24/EU и руководством OECD, на 121 вручную размеченном
   флаге даёт **6.6 % строгой точности** на корпусе 223-ФЗ — обязательные
   нормы российского законодательства систематически интерпретируются как
   индикаторы ограничения конкуренции.

2. **Белый список W1–W15.** Пятнадцать категорий обязательных норм 223-ФЗ
   (антикоррупционное требование, режим СМП, национальный режим,
   стандартизированные сроки, лингвистические шаблоны и т.п.), которые
   нужно явно исключать из детекции. Интегрирован как правила исключения в
   классификационную схему v3 и в обучающий датасет дообученной модели.
   Полное описание — в `docs/codebook.md`.

3. **Перенос обучения.** FT v3a (138 обучающих примеров) распознаёт
   паттерн «Не предусмотрено» на новых лотах и идентифицирует 26 из 26
   кондитерских брендов на лоте, не входившем в обучение. Это не
   запоминание — это работающий доменный transfer learning на меньшем
   масштабе, чем у больших frontier-моделей.

### Архитектура каскада

```
Лот документация
    ↓
[Этап 1] Парсинг и сегментация (PDF/DOCX/RTF/XLSX → 7 разделов)
    ↓
[Этап 2] rule_detector_v2 (R1–R6, regex, $0)
    ↓ ловит: пустые критерии оценки, запрет «эквивалент», ФИО+ООО
    ↓
[Этап 3] FT gpt-4.1 v3a с фильтрацией по меткам ($15/200 лотов, ~5 с/лот)
    ↓ ловит: бренды в ТЗ, узкие тех. спецификации, скрытое нацеливание
    ↓
[Этап 4] Дедупликация по (лот, метка, фрагмент) и сборка risk-report
```

## Артефакты и репо

- **Корпус** — 1018 уникальных лотов через программный интерфейс Marker
  Interfax, 1423 файла после конверсии .doc → .docx (`docs/api-discovery.md`).
- **Разметка** — 330+ вручную верифицированных экземпляров флагов + 13
  вердиктов вторичной разметки LOW-лотов (`data/labeled/`, gitignored).
- **Дообученные модели** — `ft:gpt-4.1-2025-04-14:vghb:tender-anomaly-{v1,v2,v3a}` и
  `ft:gpt-4.1-mini-2025-04-14:vghb:tender-anomaly-v3a-mini` через OpenAI Fine-tuning API.
- **Графики** — `docs/charts/` (7 публикационных PNG).

## Технологический стек

| Слой | Инструменты |
|---|---|
| Парсинг | `pdfplumber`, `PyMuPDF`, `python-docx`, MS Word COM (для .doc) |
| Сегментация | regex + 7 содержательных разделов (ТЗ / критерии / договор и т.д.) |
| LLM | OpenAI Chat Completions API (gpt-4o-mini, gpt-4o, gpt-4.1, gpt-5.5), Anthropic Claude (для редактуры текста) |
| Дообучение | OpenAI Fine-tuning API (SFT на gpt-4.1) |
| Структурированный вывод | pydantic + structured output |
| Правила | Python regex (`scripts/rule_detector_v2.py`) |
| Оценка | Ручная верификация 330+ flag-instances + κ-Коэна не измерялся (один аннотатор) |
| Графики | matplotlib (скрипты в `scripts/` для пересборки) |

Python 3.11+, виртуальное окружение через `venv`. Установка — `pip install -e ".[dev]"`.
